# 균형 상태 표시 기능 (Balance Display)

실시간 센서 데이터를 사용자에게 직관적으로 보여주는 핵심 UI 기능입니다.

---

## 목적

- 현재 기울기 상태를 즉시 이해할 수 있도록 시각화합니다.
- 숫자(각도), 색상(상태), 방향(좌/우·앞/뒤)을 동시에 제공합니다.
- 캘리브레이션/개선 가이드 기능과 자연스럽게 연결됩니다.

---

## 화면 구성

### 1) 중앙 상태 원
- 현재 균형 상태를 나타내는 메인 시각 요소
- 기울기 방향에 따라 원의 중심 또는 내부 마커가 이동

### 2) 각도 텍스트
- 예시: `+3.2°`, `-0.8°`
- 소수점 1자리 고정(권장)

### 3) 상태 라벨
- `완벽`, `약간 기울어짐`, `심하게 기울어짐`

### 4) 보조 정보
- 마지막 업데이트 시간
- 센서 신뢰도(선택): 높음/보통/낮음

---

## 상태 판정 규칙

`docs/architecture/balance-calculation.md` 기준값을 사용합니다.

| 각도 범위 | 상태 | 색상 |
|---|---|---|
| -1° ~ +1° | 완벽 | 🟢 |
| -5° ~ -1°, +1° ~ +5° | 약간 기울어짐 | 🟡 |
| < -5°, > +5° | 심하게 기울어짐 | 🔴 |

---

## 데이터/이벤트 흐름

1. 센서 이벤트 수신 (`sensors/`)
2. 이동 평균 등 필터 처리 (`architecture/sensor-data-processing.md`)
3. 기울기 계산 (`calculations/`)
4. 상태 판정(임계값 비교)
5. UI 반영(원, 각도, 색상, 텍스트)

권장 업데이트 주기:
- 센서 입력은 고주파로 수신
- UI 렌더는 초당 20~30회 범위로 제한(배터리/발열 고려)

---

## UX 규칙

- 급격한 점프 방지: 값 변화가 큰 경우 짧은 애니메이션(100~200ms) 적용
- 접근성: 색상만으로 상태를 구분하지 않고 텍스트 라벨 동시 제공
- 안정 상태 피드백: 2초 이상 `완벽` 유지 시 체크 아이콘 노출(선택)

---

## 예외/오류 처리

### 센서 미지원
- 메시지: `이 기기에서는 필요한 센서를 사용할 수 없습니다.`
- 기능 버튼 비활성화 + 설정 안내 링크 제공

### 센서 권한/초기화 실패
- 재시도 버튼 제공
- 반복 실패 시 진단 화면(`testing-guide.md`의 센서 점검 시나리오 참고)

### 데이터 품질 저하
- 신뢰도 `낮음` 표시
- 캘리브레이션 재실행 CTA 제공
