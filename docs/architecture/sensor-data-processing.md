# 센서 데이터 처리 (Sensor Data Processing)

Balance Fit이 센서 데이터를 어떻게 수집하고 처리하는지 설명합니다.

---

## 센서 선택

Balance Fit은 두 가지 센서를 사용합니다:

### 1. 가속도계 (Accelerometer)
- **측정 대상**: 기기의 기울기와 무게 분포
- **활용**: 물체가 어느 방향으로 기울어져 있는지 감지
- **예시**: 왼쪽으로 기울면 가속도계 값이 변함

### 2. 자이로스코프 (Gyroscope)
- **측정 대상**: 기기의 회전 속도와 방향
- **활용**: 물체의 흔들림 감지 및 안정성 판단
- **예시**: 물체가 흔들리면 자이로스코프가 회전을 감지

---

## 데이터 수집 주기

### 센서 데이터 읽기 빈도
- 📥 **원시 데이터**: 초당 100회 이상 (매우 빠름)
- 🎯 **사용 데이터**: 1초마다 한 번 또는 균형 상태 변화 시 기록

### 왜 모든 데이터를 저장하지 않을까?

**문제점:**
```
초당 100개 센서 데이터 × 3600초 (1시간) = 360,000개 저장
스마트폰 저장소 부족 및 배터리 소모
```

**해결책:**
```
센서는 계속 읽음 (실시간 균형 표시 필요)
↓
데이터 필터링 (1초마다 1개만 저장, 또는 변화 감지 시 저장)
↓
스마트폰 자원 효율화 + 정확도 유지
```

## 기록 기능 (Recording Feature)

### 목적
사용자가 **언제 측정 데이터를 저장할지 선택**할 수 있도록 합니다.

### 동작 방식

**기록 OFF (기본값)**
```
센서 데이터 읽음
    ↓
실시간 균형 표시 (화면에 보여줌)
    ↓
데이터 저장 안 함
```

**기록 ON (사용자 선택)**
```
센서 데이터 읽음
    ↓
실시간 균형 표시 (화면에 보여줌)
    ↓
필터링된 데이터를 데이터베이스에 저장
    ↓
과거 기록에 추가됨
```

### 화면 표시
앱 상단에 토글 버튼 (ON/OFF):
```
┌─────────────────────┐
│ 기록: [OFF] → [ON]  │  ← 버튼
├─────────────────────┤
│      균형 표시 화면  │
└─────────────────────┘
```

### 사용 예시

**상황 1: 일반 사용**
```
기록 OFF 상태로 앱 실행
→ 물체의 균형을 확인만 함
→ 데이터는 저장 안 됨
```

**상황 2: 데이터 기록 및 분석**
```
1. "기록" 버튼을 ON으로 설정
2. 여러 물체의 균형을 측정
3. 모든 측정 데이터가 데이터베이스에 저장됨
4. 나중에 "기록 보기" 메뉴에서 과거 측정 데이터 조회 가능
```

**상황 3: 산업용 전문가 사용**
```
1. "기록" ON으로 설정
2. 하루종일 여러 물건 균형 측정
3. 모든 데이터 저장됨
4. 저녁에 분석 리포트 생성 가능
```

## 데이터 저장 정책

### 언제 저장할까?

**옵션 A: 시간 기반 저장 (추천) ⭐**
- 1초마다 균형 데이터 1개씩 저장
- 장점: 단순하고 구현하기 쉬움
- 단점: 변화 없는 데이터도 저장됨

**옵션 B: 변화 기반 저장**
- 균형 상태가 변할 때만 저장
  - 예: 초록 → 노랑으로 변했을 때
  - 예: 각도가 1도 이상 변했을 때
- 장점: 저장 데이터 최소화
- 단점: 구현이 복잡함

**추천**: 처음엔 옵션 A로 시작, 나중에 필요시 B로 변경

### 데이터 삭제 정책

**저장된 데이터는 절대 자동 삭제하지 않음**
- 사용자가 수동으로만 삭제 가능
- 왜? → 과거 기록을 언제든 확인해야 하기 때문

**선택적 삭제 메뉴 (추후 구현)**
```
"기록" 메뉴
├─ 모든 기록 보기
├─ 기록 검색
├─ 기록 삭제 (선택)  ← 사용자가 필요시만 삭제
└─ 기록 내보내기
```

## 데이터 흐름도

```
스마트폰 센서
    ↓
센서 데이터 수집 (초당 100회)
    ↓
데이터 필터링 (1초마다 1개씩 또는 변화 감지)
    ↓
실시간 균형 계산 & 화면 표시
    ↓
기록 ON인가?
├─ YES → 데이터베이스 저장
└─ NO  → 저장 안 함
    ↓
사용자가 과거 기록 조회 가능
```

## 센서 데이터 구조

각 센서 데이터는 다음 정보를 포함합니다:

```
{
  timestamp: "2026-06-13 10:43:00",    // 측정 시간
  accelerometer: {
    x: 9.8,    // X축 가속도 (m/s²)
    y: 0.2,    // Y축 가속도
    z: 0.1     // Z축 가속도
  },
  gyroscope: {
    x: 0.05,   // X축 회전 속도
    y: -0.03,  // Y축 회전 속도
    z: 0.01    // Z축 회전 속도
  },
  calculatedAngle: 2.5,    // 계산된 균형 각도 (도)
  stabilityStatus: "STABLE" // 안정성 상태
}
```

## 스마트폰 자원 고려사항

### 배터리 소모
- 센서 계속 실행: 배터리 소모
- 해결책: 사용자가 필요 없을 때 앱을 끄면 센서도 중지

### 저장소 용량
- 1시간 사용 시: 약 3,600개 데이터 저장 (약 1MB)
- 나중에 삭제 메뉴로 관리

### 메모리 사용
- 메모리에는 최근 1초 데이터만 보관
- 이전 데이터는 데이터베이스에 저장

## 캘리브레이션과의 관계

센서 데이터는 캘리브레이션과 밀접한 관계가 있습니다:

```
캘리브레이션 실행
    ↓
"평탄한 표면에서 센서 기준값 저장"
    ↓
이후 모든 센서 데이터는 이 기준값과 비교
    ↓
기준값 - 현재값 = 기울기 각도
```
