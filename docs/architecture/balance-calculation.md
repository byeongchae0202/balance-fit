# 균형 계산 로직 (Balance Calculation)

센서 데이터로부터 물체의 기울기 각도를 계산하는 방법을 설명합니다.

---

## 기본 원리: 중력과 기울기

### 중력의 역할
스마트폰에는 항상 **중력이 작용**합니다:

**평평한 표면 (기울기 0도)**
```
        스마트폰
        ┌─────┐
        │  ▼  │  ← 중력이 수직으로 아래로 작용
        └─────┘
        ▲▲▲▲▲
        책상 (수평)

센서 값: X=0, Y=0, Z=9.8 (모든 중력이 Z축에만)
```

**앞으로 기울어진 상태 (기울기 약 15도)**
```
          스마트폰
          ┌─────┐
         /  ▼   │  ← 중력이 비스듬히 작용
        /       │
       / ────── /
      /책상
      
센서 값: X=0, Y=2.5, Z=9.4 (중력이 Y축과 Z축에 분산)
```

### 핵심 개념
**물체가 기울어지면 → 스마트폰도 기울어짐 → 센서로 감지됨**

---

## 계산 방법

### 단계 1: 캘리브레이션 (기준점 설정)

앱을 처음 실행하거나 "센서 캘리브레이션" 버튼을 누르면:

```
1. 스마트폰을 완전히 평평한 표면에 놓음
2. 3초 대기
3. 현재 센서 값을 "기준값"으로 저장
   - 기준값_X = 0
   - 기준값_Y = 0
   - 기준값_Z = 9.8 (중력 가속도)
```

### 단계 2: 기울기 각도 계산

물체 위에 스마트폰을 놓고 센서 데이터를 받으면:

**만약 물체가 앞으로 기울어진 경우:**
```
현재 센서 값:
- X = 0.5
- Y = 2.5
- Z = 9.4

기울기 각도 계산:
angle_Y = arctan2(Y값, Z값) = arctan2(2.5, 9.4) ≈ 15도
```

**만약 물체가 옆으로 기울어진 경우:**
```
현재 센서 값:
- X = 2.5
- Y = 0.5
- Z = 9.4

기울기 각도 계산:
angle_X = arctan2(X값, Z값) = arctan2(2.5, 9.4) ≈ 15도
```

---

## arctan2() 함수란?

### 쉬운 설명

**arctan2 = 두 숫자를 나누었을 때 각도를 구하는 함수**

예시:
```
arctan2(1, 1) = 45도
arctan2(2.5, 9.4) = 15도
arctan2(9.4, 0) = 90도
```

### 물리학적 의미

중력 벡터의 방향을 2D 각도로 변환:

```
Z축(위아래)
   ▲
   │         ←→ X축(좌우) 또는 Y축(앞뒤)
   │
   └──────→

arctan2(수평값, 수직값) = 기울기 각도
```

### 왜 arctan2를 사용할까?

일반적인 나눗셈(Y/Z)을 하면 **마이너스 각도를 표현 못합니다**:
```
나눗셈만 사용: 3 / 9 = 0.33... (각도 계산 불가)

arctan2 사용: arctan2(3, 9) = 18도 ✓
arctan2(-3, 9) = -18도 ✓ (반대 방향도 표현 가능)
```

## 실제 계산 공식

### 공식 1: 앞뒤 기울기 (Y축)
```
angle_pitch = arctan2(Y값, Z값)
```

### 공식 2: 좌우 기울기 (X축)
```
angle_roll = arctan2(X값, Z값)
```

### 공식 3: 3D 기울기 (최종)
```
최종 기울기 각도 = √(angle_pitch² + angle_roll²)

또는 사용자 관점에 따라:
- 앞뒤만 보기: angle_pitch 사용
- 좌우만 보기: angle_roll 사용
```

## 코드 수준의 설명 (참고)

프로그래밍에서는 이렇게 계산됩니다:

```kotlin
// Kotlin 코드 예시 (실제 구현)

// 1단계: 센서 데이터 받기
val accelerometerData = sensorManager.getAccelerometerData()
val xValue = accelerometerData.x
val yValue = accelerometerData.y
val zValue = accelerometerData.z

// 2단계: 각도 계산 (arctan2 함수 사용)
val anglePitch = Math.atan2(yValue, zValue) * 180 / Math.PI
val angleRoll = Math.atan2(xValue, zValue) * 180 / Math.PI

// 3단계: 최종 기울기 각도
val finalAngle = Math.sqrt(anglePitch * anglePitch + angleRoll * angleRoll)

// 4단계: 화면에 표시
displayAngle(finalAngle)
```

## 실제 예시

### 예시 1: 가벼운 기울기

```
물체가 오른쪽으로 약간 기울어짐

센서 데이터:
- X = 1.0
- Y = 0.2
- Z = 9.7

계산:
angle_roll = arctan2(1.0, 9.7) = 5.9도

결과: "오른쪽으로 약 6도 기울어짐"
화면: 🔴 (빨간색), "-6°"
```

### 예시 2: 중간 정도의 기울기

```
물체가 앞으로 기울어짐

센서 데이터:
- X = 0.1
- Y = 3.2
- Z = 9.3

계산:
angle_pitch = arctan2(3.2, 9.3) = 18.8도

결과: "앞으로 약 19도 기울어짐"
화면: 🟡 (노란색), "+19°"
```

### 예시 3: 완벽한 균형

```
물체가 완벽히 평평함

센서 데이터:
- X = 0.0
- Y = 0.0
- Z = 9.8

계산:
angle_roll = arctan2(0.0, 9.8) = 0도
angle_pitch = arctan2(0.0, 9.8) = 0도

결과: "완벽한 균형"
화면: 🟢 (초록색), "0°"
```

## 정확도 개선

### 노이즈 제거 (Smoothing)

센서는 항상 약간의 오류가 있습니다:
```
센서 값이 계속 흔들림:
8.2 → 8.1 → 8.3 → 8.2 → 8.1 ...

해결: 여러 값들의 평균을 계산
(8.2 + 8.1 + 8.3 + 8.2 + 8.1) / 5 = 8.18
```

### 이동 평균 필터 (Moving Average)

```
최근 5개 센서 데이터의 평균값을 사용

1번째 측정: Z = 9.8
2번째 측정: Z = 9.7  → 평균 = (9.8 + 9.7) / 2 = 9.75
3번째 측정: Z = 9.9  → 평균 = (9.8 + 9.7 + 9.9) / 3 = 9.8
4번째 측정: Z = 9.6  → 평균 = (9.8 + 9.7 + 9.9 + 9.6) / 4 = 9.75
5번째 측정: Z = 9.8  → 평균 = (9.8 + 9.7 + 9.9 + 9.6 + 9.8) / 5 = 9.76
```

## 균형 판정 기준 (constants에서 관리)

계산된 각도를 기반으로:

```
각도 범위          상태      색상    해석
─────────────────────────────────────────
-1° ~ +1°        완벽      🟢 초록   "균형잡힘"
-5° ~ -1°        약간      🟡 노랑   "왼쪽으로 약간 기울어짐"
+1° ~ +5°        약간      🟡 노랑   "오른쪽으로 약간 기울어짐"
< -5°            심함      🔴 빨강   "왼쪽으로 많이 기울어짐"
> +5°            심함      🔴 빨강   "오른쪽으로 많이 기울어짐"
```

## 참고 자료

- **중력 가속도**: 약 9.8 m/s² (스마트폰에서도 동일)
- **arctan2 함수**: 역탄젠트 함수 (각도를 역산하는 함수)
- **Radian과 Degree**: 라디안 단위로 나온 값을 도(°)로 변환하려면 180/π를 곱함
