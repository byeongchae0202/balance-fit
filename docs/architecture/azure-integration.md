# Azure 클라우드 통합 (Azure Integration)

Balance Fit의 측정 데이터를 Azure 클라우드에 저장하고 관리하는 방법을 설명합니다.

---

## Azure 통합의 목적

- 📱 **여러 기기에서 데이터 동기화**: 같은 계정으로 로그인하면 모든 기기에서 데이터 접근
- ☁️ **데이터 안전 보관**: 클라우드에 백업되어 기기 분실 시에도 데이터 복구 가능
- 👥 **팀 협력**: 산업용 전문가들이 팀 내에서 측정 데이터 공유 및 분석
- 📊 **분석 및 리포트**: 시간에 따른 추세 분석, 통계 자동 생성
- 🔒 **보안**: Azure의 암호화 및 인증 기능으로 데이터 보호

---

## 데이터 저장 구조

### Azure에 저장되는 정보

**측정 데이터 (Measurement Record)**
```
{
  id: "measurement_001",
  userId: "user_12345",
  timestamp: "2026-06-13 10:50:32",
  profile: "indoor",
  
  // 센서 데이터
  sensorData: {
    angle_pitch: 2.5,
    angle_roll: 1.8,
    final_angle: 3.1,
    temperature: 22.5,
    stability_status: "STABLE"

  // 균형 상태
  balanceStatus: "yellow",  // green, yellow, red
  rawSensorValues: {
    x: 1.8,
    y: 2.5,
    z: 9.4
  },
  
  // 캘리브레이션 참고
  calibrationProfile: "indoor_2026-06-13",

  // 메타데이터
  deviceInfo: {
    deviceId: "device_abc123",

```

**사용자 프로필 (User Profile)**
```
{
  userId: "user_12345",
```

## 데이터 동기화

### 동기화 플로우

```
1️⃣ 로컬 측정
   ↓
2️⃣ 기기에 저장 (즉시)
   ↓
3️⃣ 인터넷 연결 확인
   ├─ Wi-Fi 또는 4G/5G 연결
   └─ 백그라운드에서 자동 동기화
   ↓
4️⃣ Azure로 전송
   ↓
5️⃣ 서버 저장 완료
   ↓
6️⃣ 모든 기기에서 접근 가능
```

### 오프라인 모드

인터넷이 없을 때:
```
측정 데이터 → 로컬 저장소 (큐에 저장)
   ↓
인터넷 연결 감지
   ↓
자동으로 큐에 있는 데이터를 Azure로 전송
   ↓
전송 완료 후 큐 비우기
```

**개인만 접근 가능:**
```

├────────────────────────────────┤

│ 2026-06-12 15:20 | 각도: 5.2° │
│ ...                            │
│                               │
│ [내보내기] [삭제] [분석]       │

```

**기능:**
- 측정 기록 조회
- CSV/PDF로 내보내기
- 개인 통계 보기 (최근 1주일, 1개월, 1년)
- 선택한 기록 삭제

## 팀 협력 (Professional/Team)

### 팀 구성

**팀 리더 (Team Leader):**
```
역할:
- 팀 생성 및 관리
- 멤버 초대/제거
- 공유 캘리브레이션 프로필 관리
- 팀 리포트 생성 권한

화면:
┌────────────────────────────────┐
│ 팀 대시보드                    │
├────────────────────────────────┤
│ 팀 이름: Balance Team          │
│ 멤버: 5명                      │
│                               │
│ 최근 활동:                     │
│ · user_001: 오후 2시 측정     │
│ · user_002: 오후 1시 측정     │
│                               │
│ [멤버 관리] [리포트] [설정]    │
└────────────────────────────────┘
```

**팀 멤버 (Team Member):**
```
역할:
- 개인 측정 데이터 기록
- 팀 공유 데이터 접근
- 팀 리포트 확인

화면:
┌────────────────────────────────┐
│ 팀 멤버 대시보드               │
├────────────────────────────────┤
│ 팀 이름: Balance Team          │
│ 리더: Hong Gildong             │
│                               │
│ 팀 공유 데이터:                │
│ · 실내 프로필 (92% 신뢰도)   │
│ · 실외 프로필 (88% 신뢰도)   │
│                               │
│ [데이터 공유] [리포트 확인]    │
└────────────────────────────────┘
```

### 팀 데이터 공유

**공유 프로필:**
```
팀 리더가 설정한 캘리브레이션 프로필을 모든 멤버가 사용 가능

팀 리더:
"실외 현장 프로필" 생성 (정확도 90%)
   ↓
모든 팀 멤버가 접근 가능
   ↓
같은 기준으로 현장 측정 가능
```

**공유 데이터 분석:**
```
리더: "팀의 지난주 측정 데이터로 리포트 생성"
   ↓
Azure에서 모든 멤버의 데이터 집계
   ↓
자동 통계 생성
   ↓
"금주 평균 기울기: 3.2°, 성공률: 94%"
```

## 데이터 보안

### 인증

**로그인 방법:**
```
1️⃣ 이메일 + 비밀번호
2️⃣ 소셜 로그인 (Google, Apple)
3️⃣ 생체인증 (지문, 얼굴)
```

### 데이터 암호화

```
기기 ← (HTTPS 암호화) → Azure 서버

저장: AES-256 암호화
전송: TLS 1.3 암호화
```

### 접근 권한 관리

```
개인 데이터:
- 본인만 접근 가능

팀 공유 데이터:
- 팀 멤버만 접근 가능
- 팀 리더: 전체 관리 권한
- 팀 멤버: 읽기 권한

삭제:
- 사용자가 요청하면 30일 후 완전 삭제
```

## 데이터 보관 정책

### 자동 보관

개인 사용자 기본 설정:
```
측정 데이터: 1년 보관 (365일)
캘리브레이션: 무제한 보관
사용자 프로필: 계정 삭제 시까지 보관
```

팀 데이터:
```
팀 공유 데이터: 무제한 보관
팀 리포트: 3년 보관
멤버 기록: 팀이 존재하는 동안 보관
```

### 수동 관리

```
┌────────────────────────────────┐
│ 데이터 관리                    │
├────────────────────────────────┤
│ [데이터 내보내기]              │
│ CSV, JSON, PDF 형식 가능      │
│                               │
│ [데이터 삭제]                  │
│ 특정 기간 또는 선택한 항목    │
│                               │
│ [계정 삭제]                    │
│ 모든 데이터 30일 후 삭제      │
└────────────────────────────────┘
```

## 네트워크 상황별 동작

### 좋은 Wi-Fi 연결
```
측정 완료 후 즉시 Azure로 동기화 (< 1초)
```

### 느린 모바일 네트워크
```
측정 완료 후 백그라운드에서 천천히 동기화
사용자는 계속 앱 사용 가능
동기화 완료 시 알림 (선택사항)
```

### 인터넷 없음
```
로컬 저장소에 임시 저장
다음 인터넷 연결 시 자동 동기화
```

## API 엔드포인트 (개발자용 참고)

```
측정 데이터 저장:
POST /api/measurements

측정 데이터 조회:
GET /api/measurements?userId=xxx&startDate=xxx&endDate=xxx

팀 정보 조회:
GET /api/teams/{teamId}

팀 멤버 초대:
POST /api/teams/{teamId}/members

리포트 생성:
POST /api/reports/generate
```
