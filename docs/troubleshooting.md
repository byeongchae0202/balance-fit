# 문제 해결 가이드 (Troubleshooting)

Balance Fit 개발/테스트 중 자주 발생하는 문제와 해결 방법을 정리합니다.

---

## 1) Android Studio 관련

### 증상
- 프로젝트 열기/동기화가 매우 느림
- Gradle Sync 실패

### 점검
1. Android Studio와 Gradle 플러그인 버전 확인
2. 네트워크 상태 확인(사내망/프록시 포함)
3. `File > Invalidate Caches` 후 재시작

### 해결
- 캐시 초기화 후 재동기화
- 필요 시 `build/`, `.gradle/` 정리 후 재시도

---

## 2) 앱 실행 실패

### 증상
- 에뮬레이터/실기기에서 앱이 바로 종료됨

### 점검
1. Logcat에서 첫 번째 예외 확인
2. Android SDK/빌드 툴 설치 상태 확인
3. 디버그 서명/기기 연결 상태 확인

### 해결
- 누락된 SDK 설치
- 최소 SDK/타깃 SDK 호환성 확인
- 앱 데이터 삭제 후 재실행

---

## 3) 센서 값이 이상함

### 증상
- 각도가 계속 흔들림
- 평평한 표면에서도 0° 근처가 아님

### 점검
1. 캘리브레이션 최신 상태 확인
2. 기기가 흔들리는 환경인지 확인
3. 케이스/자석 등 외부 영향 확인

### 해결
- 캘리브레이션 재실행
- 안정된 표면에서 재측정
- 반복 흔들림 시 테스트 모드에서 원시 값 확인

---

## 4) 캘리브레이션 실패

### 증상
- 검증 단계에서 실패가 반복됨

### 점검
1. 샘플 수집 중 백그라운드 전환 여부 확인
2. 센서 이벤트 누락/지연 여부 확인
3. 저장 단계 오류 로그 확인

### 해결
- 캘리브레이션 중 앱 포그라운드 유지
- 수집 시간 또는 샘플 수 조정
- 저장 실패 원인(권한/저장소 상태) 수정 후 재시도

---

## 5) 개선 가이드 문구가 부정확함

### 증상
- 기울기와 반대 방향 안내가 나오지 않음
- 이동량이 과도하거나 너무 작음

### 점검
1. 각도 부호 처리(+, -) 로직 확인
2. 단계 매핑 테이블 임계값 확인
3. 최근 캘리브레이션 값 반영 여부 확인

### 해결
- 부호/방향 매핑 테스트 케이스 추가
- 임계값 재조정
- 캘리브레이션 데이터 갱신 후 재검증

---

## 6) 문서와 실제 동작이 다름

### 점검
1. 먼저 `docs/testing/testing-guide.md` 시나리오로 재현
2. 재현 가능하면 문서 또는 구현 중 기준 소스를 확정

### 해결
- 동작이 맞다면 문서 수정
- 문서가 맞다면 구현 수정
- 수정 후 관련 문서 링크 동기화
